  1. What is it?
    • This program provides the functions shown below.
      1. cobcc command converts the call statements for DB Access from the IO statements in COBOL source.
      2. "libXXXClient.so" shared object provides functions for DB access.

        Each of functions corresponds to the IO statement of COBOL source.

  5. Generate
    • cobpp

      creates COBOL source code which includes CALL statements to access db from original code.

      • format

        cobpp infile outfile

        • infile
        • outfile
      • options
        • -i/--informatfix or other input source code format. default is fix
        • -o/--outformatfix or other output source code format default is fix
        • -r/--initrowinitial row number default is 000010
        • -c/--increaseincrease row number default is 10
        • -a/--acmconsts_filelocation of ACMCONSTS.CBL
        • -e/--expand_copyexpand file on source directed by copy statement
        • -l/--codegenaratorlisteners eventlistener class names separated by :
        • -x/--customcode generatorclasscustom class for generating souce code
        • -s/--charset charset of source code
        • -t/--tcp generate for client of sqlnetserver
        • -j/--jni generate for client with JNI
        • -b/--subprogram generate for subprogram/don't insert initialize functions
        • -h/--help show help message.
  6. Others
    • cob2meta

      creates metadata file from COBOL source.

      cob2meta infile outfile

      • options
        • -i/--informat fix or other input source code format. default is fix
        • -s/--charset charset of source code
    • acm2seq

      exports DB records to sequential file.

      acm2seq acmfile outfile

      • options
        • -m/--metafileconfiguration file of record layout. default is /opt/sa4cob2db/conf/metafile.xml
        • -l/--lineouttrue/false outfile is line file. default is true
        • -q/--sqluse custom sql
        • -a/--sqlinuse custom sql from standard in.
    • seq2acm

      imports to DB records from sequntial file.

      acm2seq acmfile infile

      • options
        • -m/--metafileconfiguration file of record layout. default is /opt/sa4cob2db/conf/metafile.xml
        • -l/--lineintrue/false infile is line file. default is true
        • -e/--extendtrue/false open acmfile mode is extend. default is false.
        • -c/--commitcommit count. Default is auto commit.
