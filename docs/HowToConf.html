  2. Configuration File(DTD)
    1. meatadataset
      1. Elements
        2. entire settings
        1. jdbcdriverurl
          2. JDBC Driver URL.
        2. jdbcdatabaseurl
          3. JDBC Database URL
        3. jdbcusername
          4. JDBC Database Username.
        4. jdbcpassword
          5. JDBC Database Password.
        5. address
          6. Hostname or Address which SQLNetServer accepts. Comma(,) Seperated names or addresses.
          default value is localhost.
        6. port
          7. Port number which SQLNetServer accepts.
          default value is 12345.
        7. maxsessions
          8. Maximum session count. It means unlimited count if 0 set to this value.
        8. log
          9. Logging configuration File name.
        9. authfile
          10. Password File name.
        10. adminfile
          11. Administrators Password File name..
        11. backlog
          12. Backlog of Server Socket that SQLNetServer uses.
        12. sessionlisteners
          13. Classes process session events.
          Class names separated by ":"
        13. serverlisteners
          14. Classes process server events.
          Class names separated by ":"
      2. otherofset
        3. Class to handle other nodes nor in the entire settings in metadata. This class has to implement MetaDataSetNodeProcessor interface.
        Attributes:name:Name of node
        Element:Classname
      3. otherofmeta
        4. Classes to handle node that is not defined. This class has to implement         MetaDataNodeProcessor interface.
      4. metadata:Each metadata/file settings
        5. Attributes
        1. name
          2. External file name or Metadata name.
          YYYY of 'SELECT XXXX ASSIGN TO "YYYY"'.
        2. bufinit
          3. Initial buffering record count for sequential read.
          'READ NEXT' waits until count of buffered records reached this value.
        3. bufmin
          4. Minimum buffering record count for sequential read.
          'READ NEXT' waits until count of buffered records reached this value.
        4. bufmax
          5. Maximum buffering record count.
          Maximum number of records to be buffered in the sequential read operation.
        5. encode
          6. Character encoding of record.
        6. customClassName
          7. Class of order to the operation of its own non-built-in.
          This class has to           CobolRecordMetaData interface.
        Element
        1. statement
          2. SELECT statement
          You need to sort the result set by "ORDER BY" if the file is read Random/Dynamic.
        2. sqlcolumn
          3. Column. build-in object.
          Attributes
          1. name
            2. Column name.
          2. type
            3. Data type.
            ValueCobol TypeDescription
            19integer
          3. start
            4. 開始位置。0で始まります。
          4. length
            5. 列長。バイト数です。
          5. signed
            6. 列長。バイト数です。
          6. decimal
            7. 列長。バイト数です。
          7. usage
            8. 列長。バイト数です。
          8. format
            9. 書式文字列。数値型などのデータの整形などに利用します。
            組み込みの型では以下に対応するフォーマット文字列を利用します。
            データ型クラス
            数値型java.text.DecimalFormat
            日付型java.text.SimpleDateFormat
          9. originalColumnName
            10. SQL結果セットのフィールド名。
          10. defaultString
          11. readIgnore
            12. 読み込み(read/select)時に無視するかどうか
            設定しない場合は"false"。
            セットした値動作
            false結果セットの値でコボル側データを更新する。
            true結果セットの値でコボル側データを更新しない。
          12. rewriteIgnore
            13. 更新(rewrite/update)時に無視するかどうか
            設定しない場合は"false"。
            セットした値動作
            falseコボル側データで結果セットの値を更新する。
            trueコボル側データで結果セットの値を更新しない。
          13. writeIgnore
            14. 挿入(write/insert)時に無視するかどうか
            設定しない場合は"false"。
            セットした値動作
            falseコボル側データで結果セットの値を更新する。
            trueコボル側データで結果セットの値を更新しない。
        3. customcolumn
          4. 組み込みの列では実現できない独自の動作をする列を定義します。
          属性
          1. name
            2. 列名。
            sqlcolumn/customcolumnに関わらずmetadata内で一意に識別可能な名前を仕様してください。
            (1ファイル内で重複しないようにしてください。)
          2. classname
            3. 列の動作を規定するクラス名。ここに指定されるクラスは            CobolColumnを実装する必要があります。
            設定ファイルではクラスパスを設定・更新しません。
            クラスパスは起動スクリプトなどに記述しておいてください
          3. constructor
            4. この列のクラスのオブジェクトを作成するためのコンストラクタを選択します。
            値引数
            1引数なし
            2メタデータこの列を含むメタデータ/ファイルレイアウトに相当するmetadataオブジェクト
            3メタデータこの列を含むメタデータ/ファイルレイアウトに相当するmetadataオブジェクト
            ノードこの列を定義しているnodeオブジェクト
        4. keycolumn
          5. キー項目の列名(sqlcolumn/customcolumnのname属性の値)
        5. indexfile
          6. START命令などで使用するための索引を定義します。
          属性
          1. filename
            2. 事前に作成したレイアウト/メタデータ/ファイルを指定します。
          2. indexname
            3. COBOLプログラムが使用するための索引名を指定します。
          ノード
          ファイルとインデックスの列の関連付けを行います。
          1. index2file
            2. インデックスの列をファイルのキー列に割り当てます。
            属性
            1. filecolumn
              2. ファイル側の列
            2. indexcolumn
              3. インデックス側の列
          2. file2index
            3. ファイルの列をインデックスのキー列に割り当てます。
            属性
            1. filecolumn
              2. ファイル側の列
            2. indexcolumn
              3. インデックス側の列
        6. truncate
          7. 全データを削除するSQL文を記述します。
          ファイルをOUTPUTモードでオープンしたときに実行されます。
        7. listener
          8. このメタデータのイベント処理を行うクラスを指定します。
          ここで指定するクラスはCobolFileEventListenerを実装しなければなりません。
